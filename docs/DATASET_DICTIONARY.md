# Dataset Dictionary

This dictionary documents the `daily_features` export schema from
`app/exports/daily_features_export.py`. Logical dtypes are intentionally
consumer-oriented: `decimal` values are exported as decimal strings in CSV,
dates/timestamps are ISO strings, and missing nullable values are exported as
empty CSV cells.

Common policies:

- Row grain: one row per `product_code` and `feature_date`.
- Export source: `daily_product_features` joined to `products`.
- Validation: non-empty `product_code`, non-empty `feature_date`, no duplicate
  `(product_code, feature_date)`, positive row count, SHA-256 manifest entries,
  and no secret markers in manifest.
- As-of rule: feature groups must not use source data known after
  `feature_date`; group-specific as-of columns document the cutoff.
- Missing policy: nullable feature columns use `NULL`/empty CSV values; group
  missing flag columns record known sparse coverage.
